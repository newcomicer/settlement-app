## 階段 1：Excel 前端解析（SheetJS 重寫 parse_excel）

**目標**：使用者選擇 Excel 檔後，瀏覽器端用 SheetJS 解析，結果與現有 Python 版一致。Excel 檔案不離開使用者電腦。

### Happy Path
- [ ] 上傳標準 iRunner Excel，前端解析出：組別人數、公關人數、加購數量、所有財務數字（實繳金額、ATM/超商手續費、郵寄費、晶片押金、退費、信用卡總額、折扣）
- [ ] 解析結果與 Python `parse_excel()` 回傳的 JSON 數值完全一致（逐欄位比對）
- [ ] 記錄 Excel 完整檔名（例如「2026彰化同濟盃全國馬拉松暨親子路跑報名資料-0421101313.xlsx」），並自動解析出賽事名稱與資料下載時間戳
- [ ] Excel 檔案沒有被上傳到任何伺服器（Network tab 無檔案上傳請求）

### 容易出問題的情境
- [ ] 工作表名稱不是「報名資料」而是「報名名單」等變體 → 模糊比對仍能找到
- [ ] Excel 沒有合計列（需逐訂單加總）→ 結果仍正確
- [ ] 有「免費名單」工作表 → 公關人數優先從該工作表讀取
- [ ] 親子組等多人訂單（同一訂單中有人費用為空）→ 不被誤判為公關
- [ ] 有折扣欄、非標準加購欄位 → 動態偵測正常運作

### 技術驗收
- [ ] 前端專案結構已建立（index.html + js/ + css/）
- [ ] SheetJS 透過 CDN 引入，無需 npm
- [ ] 原有 UI 功能正常（上傳後顯示組別表格、財務數字填入對應欄位）
- [ ] 不依賴 Python / Flask / 後端（可直接用 file:// 或靜態伺服器開啟）
